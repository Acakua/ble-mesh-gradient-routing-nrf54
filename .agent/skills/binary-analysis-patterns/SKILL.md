# Binary Analysis Patterns

## Use this skill when
- Analyzing executables or firmware.
- Reversing logic from binaries.
- Identifying code patterns in assembly (x86, ARM).

## Instructions
- Identify instruction patterns (branches, loops, switches).
- Recover data structures (arrays, structs).
- Map assembly back to high-level logic.
- Use tools like Ghidra or IDA Pro for decompilation review.

## Best Practices
- Document assumptions about variable types.
- Follow control flow meticulously.
- Look for common C library patterns.
